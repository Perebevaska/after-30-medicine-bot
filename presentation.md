# 1. Гонки и параллелизм 🔴

- **Дубли приёмов** — нет UNIQUE → `ON CONFLICT`
- **Потерянное обновление** → `SELECT FOR UPDATE`
- **Размножение карточек** в React-анимации
- **Потеря напоминания** при пропуске минуты

<!-- ИИ пишет счастливый путь отлично. Конкурентность, идемпотентность — не предусматривает. Видел человек, понимающий «1000 юзеров жмут одновременно». -->

---

# 2. Контекст и состояние

- Расчёт «пора пить» по времени **браузера**, не TZ юзера
- Состояние **в памяти** → не пережило рестарт
- `REDIS_URL` захардкожен в **3 местах**

**ИИ мыслит локально. Системные свойства — на человеке.**

<!-- Где живёт состояние, что переживает рестарт, единый конфиг — зона человека. ИИ охотно хардкодит и дублирует. -->

---

# 3. Архитектурные развороты

- Миграция **SQLite → PostgreSQL** → остался мёртвый код на старой БД
- Telegram **429** → пришлось вводить очередь ARQ
- `O(users×meds)` — осознанный техдолг

**ИИ не подчищает хвосты после разворотов.**

<!-- При крупных изменениях остаётся мёртвый код и рассинхрон доков. Нужен явный аудит «что устарело». -->

---

# 4. Дисциплина и дрейф

- Документация **разрасталась** → чистка/сжатие
- Фикс одного **ломал** соседнее
- Код **правдоподобен** даже когда неверен

**Главный риск — не плохой код, а правдоподобный.**

# Уроки

1. **Документация = память агента**

5. **Системные свойства держит человек**
6. **Не верь — проверяй**
