# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## About
Telegram бот для напоминаний о приёме лекарств с поддержкой гибкого расписания, статистики, экспорта в PDF и определения часовых поясов. Параллельно разрабатывается FastAPI-бэкенд для Telegram Mini App.

## Stack
- **Language**: Python 3.14
- **Framework**: python-telegram-bot 22.7 (async API)
- **Scheduler**: APScheduler 3.11.2 (задача каждую минуту, **только в bot.py**)
- **Database**: PostgreSQL 18 через psycopg 3.3 + psycopg_pool (пул соединений)
- **API**: FastAPI 0.136 + Uvicorn 0.48 (`api/`)
- **Timezone**: pytz, timezonefinder, geopy (Nominatim кэшируется как `_geolocator` на уровне модуля)
- **PDF**: fpdf2 + DejaVuSans TTF (`/usr/share/fonts/truetype/dejavu/`)
- **Container**: Podman, multi-stage Containerfile (python:3.14-slim + fonts-dejavu-core)

### Схема БД (PostgreSQL)
5 активных таблиц, все PK — `INTEGER GENERATED BY DEFAULT AS IDENTITY`:
- `users` (telegram_id BIGINT UNIQUE, username, timezone, reminder_mode, time_morning/lunch/evening/night, daily_plan_enabled, daily_plan_time, **caregiver_enabled**, created_at TEXT)
- `dependents` (user_id FK, name)
- `medications` (user_id FK, name, dosage, meal_relation, times_per_day, active, **dependent_id** FK NULL, **stock_qty** REAL NULL, **units_per_dose** REAL, **low_stock_days** INTEGER, **paused** INTEGER 0/1, created_at TEXT)
- `schedule_rules` (medication_id FK, reminder_time, frequency, interval_days, weekdays, month_day, anchor_date, dosage NULL=наследует из medications)
- `intake_log` (medication_id FK, scheduled_time, status, taken_at TEXT UTC)

**Соединение БД** (`get_connection`): psycopg_pool ConnectionPool, `row_factory=dict_row`, `init_pool(dsn)` вызывается из `bot.py:main()` и `api/main.py:lifespan`. Все таймстампы — TEXT `'YYYY-MM-DD HH:MM:SS'` UTC.
**Индексы**: `medications(user_id, active)`, `medications(dependent_id)`, `schedule_rules(medication_id)`, `intake_log(medication_id, scheduled_time)`, `intake_log(taken_at)`.

### schedule_rules — типы frequency
| frequency | поля | описание |
|-----------|------|----------|
| `daily` | — | каждый день |
| `interval` | `interval_days`, `anchor_date` | каждые N дней от anchor_date |
| `weekdays` | `weekdays` | по дням недели, '1,3,5' (пн=1, вс=7) |
| `monthly` | `month_day` | раз в месяц, N-го числа |

### API эндпоинты (FastAPI)
- `GET /health` — статус
- `GET|POST|PUT|DELETE /medications` — CRUD лекарств
- `POST /medications/{id}/pause|resume` — пауза
- `GET /today`, `POST /today/intake` — расписание на сегодня + отметка приёма
- `GET /stats/week|adherence|streak` — статистика
- `GET|PUT|POST|DELETE /medications/{id}/stock*` — учёт запаса
- `GET|POST|DELETE /dependents` — подопечные
- `GET|PUT /settings/*` — настройки пользователя
- `GET /export/plan|week|adherence|doctor` — PDF

Авторизация: `Authorization: tma <url-encoded initData>` (Telegram HMAC-SHA256, срок 24ч).
Синхронные DB-вызовы оборачиваются в `asyncio.to_thread()`.
**APScheduler запускается только в bot.py** — в API не стартуем.

### PDF builders (handlers/export.py)
Добавлены standalone функции без привязки к Telegram:
- `build_plan_pdf(telegram_id)` → `BytesIO | None`
- `build_week_stats_pdf(telegram_id)` → `BytesIO | None`
- `build_adherence_pdf(telegram_id)` → `BytesIO | None`
- `build_doctor_pdf(telegram_id, user_label)` → `BytesIO | None`

## Error Handling
- `DatabaseError` — custom exception в `database.py`
- Декоратор `@handle_db_errors` из `utils.py` — оборачивает Telegram handler-функции
- Ошибки БД → **stderr** (C5: убран `db_errors.log`, используется `StreamHandler(sys.stderr)`)
- API ошибки: 401 без валидного initData, 404 если ресурс не найден, 400 при нарушении лимитов или невалидном timezone, 422 при ошибках валидации (detail всегда строка)
- PDF генерируется в `asyncio.to_thread` чтобы не блокировать event loop

## Key Behaviors
- `init_pool()` вызывается в `bot.py:main()` до `init_db()` и в `api/main.py:lifespan`; пул идемпотентен (no-op если уже создан)
- Часовой пояс запрашивается при `/start` если не задан
- Напоминания в local time пользователя
- Режим напоминаний: `once` или `repeat` (каждые 5 минут до подтверждения, до 2 часов)
- Лимит лекарств: `MAX_MEDICATIONS_PER_USER = 10`
- `log_intake` — upsert по (medication_id, scheduled_time, дата в TZ пользователя)
- **Смена пресета мигрирует правила**: `set_user_time_preset()` переносит все `schedule_rules` пользователя с reminder_time == старое значение на новое
- **Один проход планировщика**: `get_active_schedule_rows()` — один JOIN-запрос на весь run
- **Разная дозировка**: правила А `dosage=NULL`, Б `dosage=dosage_b`; планировщик: `rule_dosage or med_dosage`
- `handle_intake_callback` парсит `status:med_id:HH:MM`
- **APScheduler только в bot.py** — api/main.py не запускает scheduler

**S3 · Затирание username** (`database.py:188`) — `ON CONFLICT DO UPDATE SET username = EXCLUDED.username`; API всегда зовёт без username → NULL на каждый запрос.
→ `SET username = COALESCE(EXCLUDED.username, users.username)`.

**S4 · Rate limiter** (`api/main.py:28-39`) — `_counters` растёт по уникальным IP (пустые ключи не чистятся); per-process (не работает с >1 воркера); за прокси (Caddy, D1) видит IP прокси.

**B5 · Нет серверной валидации правил** (`api/routers/medications.py:15-23`) — `reminder_time`, `weekdays`, `month_day`, `interval_days`, `anchor_date` принимаются без проверки.
